    宏是什么，有什么用，举个例子，现在有什么更先进的替代方案么？

---

2026-05-09 下午  |  智能体

    宏本质上是操作录像机。把一系列按键录制下来，之后随时回放，
    就像你亲手再按一遍一样。
    
    典型例子：有 100 行 CSV 数据，每行格式是 name,age,city，
    想把每行改成 { name: "xxx", age: xxx, city: "xxx" }。
    手动改一行需要 20 次按键，改 100 行就是 2000 次。用宏的话：
    
        qa          → 开始录制到寄存器 a
        （对第一行执行完整的格式转换操作）
        q           → 停止录制
        99@a        → 对剩余 99 行回放宏
    
    宏的强大之处在于它录制的是按键序列，而不是文本变换规则。
    任何你能手动做的操作，都可以录制成宏。
    
    更先进的替代：AI 自然语言操作。直接描述你想做什么，
    "把这 100 行 CSV 转成 JSON 对象格式"，AI 理解意图，生成操作，一次执行。
    不需要录制，不需要担心边界情况，意图表达比按键序列更健壮。
    在 hi 里，`?` 键触发的 AI 模式就是这个替代方案的入口。

2026-05-09 下午  |  智能体

    多光标目前不支持，实现复杂度较高，需要单独研究。
    
    命名寄存器（"a 到 "z）解决的核心问题是同时持有多份剪贴板内容。
    系统剪贴板只有一个，Vim 的默认寄存器也只有一个，
    需要同时操作多段文本时就需要命名寄存器：
    
        "ayy    → 把当前行存入寄存器 a
        "byy    → 把另一行存入寄存器 b
        "ap     → 粘贴寄存器 a 的内容
        "bp     → 粘贴寄存器 b 的内容
    
    实际场景：重构代码时把函数 A 的某段逻辑移到函数 B，
    同时把函数 B 的某段逻辑移到函数 A，两段内容需要同时持有，互不覆盖。
    
    更先进的替代：剪贴板历史。现代编辑器（VS Code、Helix）自动记录最近 N 次复制的内容，
    通过快捷键浏览历史，选择要粘贴的版本。
    比命名寄存器更自然——不需要在复制时就决定"存到哪个寄存器"，
    只需要在粘贴时选择"用哪次复制的内容"。
