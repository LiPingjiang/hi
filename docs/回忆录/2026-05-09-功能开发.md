2026-05-09 下午  |  智能体

    好，这一轮计划实现四个功能：
    `.` 重复操作、文本对象、宏录制回放、多寄存器。
    
    先说 `.` 重复操作。Vim 里最优雅的设计之一，按下 `.` 重复上一次修改。
    实现上引入 RepeatAction 枚举，记录最近一次可重复的操作类型，
    每次执行修改时把操作存到 editor.last_action，按 `.` 时读取并重新执行。
    
    Insert 模式的重复最复杂——需要记录进入 Insert 时的光标列位置、
    输入的全部文本、以及退出方式。引入 InsertSession 概念，
    进入 Insert 时开始记录，Esc 退出时保存到 last_action。

2026-05-09 下午  |  智能体

    文本对象是 Vim 操作语言的核心语法。`diw` 删除一个单词，
    `ci"` 修改引号内的内容，`da(` 删除括号及其内容。
    结构是：[operator] [i/a] [object]，需要处理三键序列。
    
    在 Normal 模式引入 pending_operator 字段，记录已按下的操作符和 i/a 修饰符，
    等第三个键（对象类型）到来时执行。支持的对象包括：
    w 单词、引号对（" ' `）、括号对（( [ { <）、s 句子。

2026-05-09 下午  |  智能体

    宏是操作录像机。`qa` 开始录制到寄存器 a，执行一系列操作，
    `q` 停止，`@a` 回放。
    
    macro_recording 字段记录当前是否在录制及录制到哪个寄存器，
    macros 是 HashMap<char, Vec<MacroKey>> 存储录制好的宏。
    录制期间每个按键都追加到键序列，回放时把键序列重新喂给按键处理逻辑，
    就像用户真的在按键一样。状态栏录制期间显示 `● REC @a` 提示。

2026-05-09 下午  |  智能体

    多寄存器：Vim 有 26 个命名寄存器（a-z），用 `"a` 前缀指定操作使用哪个。
    `"ayy` 把当前行复制到寄存器 a，`"ap` 从寄存器 a 粘贴。
    
    named_registers 是 HashMap<char, RegisterEntry>，
    active_register 记录通过 `"` 键选择的寄存器。
    每次 yank/delete 时如果 active_register 有值就写入对应寄存器，
    paste 时从对应寄存器读取。

    过程中有个小插曲：dot_repeat() 需要调用 replace_char_at_cursor 和 indent_line，
    但这两个方法是私有的。解决方案是把它们改为 pub fn。
    
    这个问题提醒了一件事：Rust 的可见性系统是有意义的约束，不是障碍。
    改成 pub 之前先想想"这个方法应该是公开的吗"——
    在这个案例里，replace_char_at_cursor 确实是合理的公开操作，改是对的。
